# Repo Audit — full (external sweep)

**Scope:** full. **Method:** synthesis of raw findings across angles 13 (product-currency),
14 (performance), 16 (dependency-currency), and carried internal angles 1/4/5/6/7/11.
**Rubric:** `.claude/rules/repo-audit.md`. This is the **first external product-currency
sweep** the 2026-06-17 inaugural audit flagged as never-yet-run (its Follow-ups, lines 57-58).

## Verdict

Repo is sound; the drift is concentrated in **shared schema/mapping files that were never
re-validated against the current product**, exactly the gap the inaugural audit predicted.
No security, auth, or data-correctness emergencies. Severity bands:

- 🔴 **Product-currency drift in shared schemas (5 high)** — three platforms each have an
  obsolete "not possible" claim that the product now supports or has changed:
  ThoughtSpot model-level NL instructions now have a GA-track API (`ai/instructions`);
  Snowflake YAML now accepts `facts[]` and `sample_values`; Databricks Metric Views are GA
  (the "Preview channel required" instruction is wrong and disruptive). Plus the duplicated
  hand-rolled invariant gate that `ts tml lint` already replaces.
- 🟠 **Doc / coverage / dependency hygiene (9 medium)** — schema files missing currency
  anchors (escaped the freshness net), an orphaned smoke test leaving `ts metadata report`
  with no live coverage, a stale ts-cli README default, no pip-audit/Dependabot gate, an
  EOL Python floor, and a round-trip smoke step that asserts nothing.
- 🟡 **Drift in prose / test identifiers / minor inefficiency (19 low)** — stale
  `/ts-profile-setup` error strings, dead `.gitignore` entries, brittle parsers, redundant
  API round-trips, and currency anchors due to be advanced after this sweep.

## Codification opportunities (manual → automated)

This sweep surfaces **two** validators worth building, both preventing silent recurrence:

1. **Currency-anchor validator** (covers 13.7, 13.11, 13.13) — fail when any
   `agents/shared/schemas/*.md` or `agents/shared/mappings/**/*.md` lacks a
   `<!-- currency: ... YYYY-MM ... -->` anchor. This single validator is why three platforms'
   schema files (snowflake-schema, all thoughtspot-*, databricks-metric-view) silently never
   got re-swept and accumulated the 5 high-severity drifts above. **Highest-leverage item in
   the sweep.**
2. **Inline-gate validator** (13.3 codification / 11.3) — flag any convert-from SKILL.md that
   contains the literal `aggregation:\s*COUNT_DISTINCT` grep or the "Pre-import validation
   gate" heading without an accompanying `ts tml lint` reference. Build only after 11.1 is
   actioned.

Plus the existing `ts tml lint` should gain an **I8 duplicate-column_id** check (11.2),
extending its codification of the model invariants.
