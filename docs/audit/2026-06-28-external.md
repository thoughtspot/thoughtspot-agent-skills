# Repo Audit — external sweep 2026-06-28

**Scope:** external (angles 13 product-currency, 14 performance, 16 dependency-currency).
**Method:** specialist sweep across angles 13 (product-currency), 14 (performance),
16 (dependency-currency). Raw findings verified via SpotterCode MCP
(`get-rest-api-reference`, `get-developer-docs-reference`) and platform documentation
(Snowflake YAML spec, Databricks YAML reference, Tableau calculation function docs).
**Rubric:** `.claude/rules/repo-audit.md`.

## Verdict

The highest-impact drift is in **Databricks Metric View** documentation: four high-severity
findings where the product has moved and our schemas/mappings will generate incorrect or
rejected YAML on current runtimes. Snowflake schema completeness has fallen behind the
YAML spec (several new fields undocumented), and three separate "untranslatable" entries in
the Tableau formula mapping contradict their own file's later sections. Dependency/CI hygiene
items from the prior audit (BL-033, target 2026-08-31) remain unactioned but are not yet
overdue.

- **High (4)** — all Databricks + Snowflake product-currency: wrong field name
  (`nulls_position` vs `null_order`), unrecognized canonical keyword (`fields:` vs
  `dimensions:`), undocumented runtime prerequisite (window `offset` requires 18.1), and a
  completely missing construct (`cardinality:` on joins). Each will cause parse errors or
  silent data loss on current product builds.
- **Medium (13)** — stale "not possible" claims now contradicted by the product (RLS in TML,
  sample_values valid, Tableau functions mapped), schema gaps (Snowflake YAML fields,
  Databricks window range types, materialization block), fabricated syntax
  (`entities:/db_connection:`), supply-chain CVE exposure, and redundant API round-trips in
  ts-dependency-manager.
- **Low (8)** — beta/preview features worth tracking, documentation-only staleness, unmapped
  Tableau functions unlikely to appear in production workbooks, and CI reproducibility gaps.
- **Confirmed clean (11)** — token caching, pagination, formula efficiency, COUNT_DISTINCT
  handling, Tableau single-pass parsing, connections column-level scoping, Tableau function
  surface stability.

## Actions taken

- **Tableau untranslatable contradictions (13.21) — FIXED in this sweep.** Removed
  `ISMEMBEROF()`, `DATETIME(expr)`, and SQL-lookup parameters from the Untranslatable
  Patterns table and added them to the "Formerly untranslatable, now mapped" list in
  `tableau-formula-translation.md`. The pipeline will no longer incorrectly omit these.
