# Repo Audit — full

**Scope:** full. **Method:** synthesis of raw findings across external angles 13
(product-currency, per platform), 14 (performance), 16 (dependency-currency), and internal
angles 1 (dead files), 4 (tools quality), 5 (ts-cli gaps), 6 (testing value), 7
(PR validation), 11 (codification). **Rubric:** `.claude/rules/repo-audit.md`.
Second full audit; diff against `docs/audit/2026-06-17-full.md` for what changed.

## Verdict

The repo remains structurally sound — CI is a verified full-repo superset of pre-commit
with admin-enforced branch protection (7.4), and no security or credential findings
surfaced anywhere. The problems concentrate in four themes:

- 🔴 **2 high — trust-breaking defects.** The impact-report deep probes swallow export
  failures but still emit coverage rows as `checked=True` — ts-dependency-manager uses
  that report to green-light destructive removals, so a failed probe is indistinguishable
  from "verified: no RLS/alert usage" (4.1). And two shipped conversion skills instruct a
  `ts tml import --file` flag that does not exist — the documented command fails outright
  (11.1).
- 🟠 **Product-currency drift, 9 medium across all four platforms** — the same pattern the
  first external sweep predicted: correct-when-written claims made obsolete by product
  movement. ThoughtSpot deprecated the batch variable endpoint we call in three places;
  Snowflake SVs now support root-level derived metrics, named filters, and free-text AI
  instructions our schema gets wrong or forbids; Databricks MV grew source forms, `using:`
  joins, and window ranges our parsers silently mishandle; Tableau's spatial and user
  function sets grew past our fail-loud lists, reopening the silent-passthrough failure
  class v0.26.0 was built to eliminate.
- 🟠 **Gate integrity, 2 medium** — the pre-commit unit-test gate false-passes when tests
  fail (`grep -q "passed"` matches `3 failed, 898 passed`), and the smoke runner's
  `REQUIRED_EXTRA_ARGS` is out of sync with argparse, turning an intended SKIP into a
  push-blocking FAIL (7.1, 6.1).
- 🟡 **Hygiene and duplication, 31 low** — stale prose, dead scaffolding, untested
  validators, and copy-paste blocks awaiting codification.
- ⚠️ **Two recurrences.** `databricks_sql` polling only on PENDING (prior 4.3) and the
  orphan `smoke_ts-metadata-report.py` (prior 6.1) were found again, unfixed — the prior
  sweep's backlog routing did not stick for them. Re-route with dated BL items this time.
