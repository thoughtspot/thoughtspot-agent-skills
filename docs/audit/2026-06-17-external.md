# Repo Audit — external (first external sweep)

**Scope:** external (angles 13 product-currency, 14 performance, 16 dependency-currency).
**Method:** product-currency verified against the SpotterCode MCP and live vendor docs
(Snowflake semantic-view YAML spec, Databricks Metric Views GA docs); performance and
dependency angles read against the repo. **Rubric:** `.claude/rules/repo-audit.md`.

This is the **first external pass** the follow-ups of the inaugural full audit called for —
currency anchors had never been validated against live product surfaces until now.

## Verdict

Repo broadly current, but product-currency has drifted on three fronts and the drift was
**structurally invisible** because shared *schema* files carry no currency anchor (only the
*mapping* files do). Severity bands:

- 🔴 **Obsolete product claims in shared schemas** — four surfaces document the product as it
  was, not as it is: ThoughtSpot model-level NL instructions now have a GA-track API
  (`ai/instructions/set|get`) the skill says doesn't exist; the Snowflake YAML spec now
  accepts `facts[]` and `sample_values` that `snowflake-schema.md` calls DDL-only / parse-error;
  Databricks Metric Views went **GA 2026-04-02** but the schema still mandates the Preview channel.
- 🟠 **Currency anchors missing on schema files + dependency hygiene gaps** — `snowflake-schema.md`
  and all `thoughtspot-*.md` / `databricks-metric-view.md` schemas have no anchor, so the
  freshness sweep can't see them. No `pip-audit`/Dependabot gate; Python floor is EOL 3.9.
  One genuine batch-export performance win in model-coach.
- 🟡 **Lower-stakes drift + cross-mapping inconsistency** — Databricks v0.1 framed as co-equal
  (docs dropped it); `window:` is Experimental but underpins a large translation block;
  COUNT_DISTINCT classification disagrees across the two reverse mappings; floor-only deps,
  unpinned CI tooling.
- 🟢 **Verified-still-correct** — `data_type`-on-metrics blocker and `relationship_type`/`join_type`
  non-support confirmed accurate against the current spec; recorded so a future sweep skips them.
- **Security:** no findings this sweep (auth/secrets out of external scope; no regressions surfaced).

## Codification opportunity (manual → automated)

The single highest-leverage fix is a **currency-anchor validator** (finding #11). Every drift
in this sweep (#1, #4, #5, #7) lived in a file with no anchor, so it was never scheduled for
re-validation. A `check_currency_anchors.py` that fails when any `agents/shared/schemas/*.md`
or `agents/shared/mappings/**/*.md` file lacks a `<!-- currency: ... YYYY-MM ... -->` anchor
would make this class of silent staleness impossible to reintroduce.

## Deduplication notes

- Three separate "no currency anchor" findings (ThoughtSpot schema, Snowflake schema,
  Databricks files) collapse into **#11** — one root cause, one validator fix. The surviving
  row carries the Snowflake severity (🟠) because that file also hosts the obsolete `facts[]`/
  `sample_values` claims.
- The Snowflake YAML-spec drift spans **#4 (facts), #5 (sample_values), #6 (labels/unique/
  cortex-search)** — three distinct fields but **one schema-file edit** resolves all of them;
  #6 is lower severity because the from-direction still parses DDL correctly.
- The model-coach NL-instructions thread spans **#1 (API exists), #2 (TML TBD), #3 (schema
  omission)** — one product surface, sequenced: settle the API-vs-TML question in #1 first.
