# Repo Audit -- full

**Scope:** full. **Method:** synthesis of raw findings across external angles 13 (product-currency,
per-platform), 14 (performance), 16 (dependency-currency), and internal angles 1 (dead files),
4 (tools quality), 5 (ts-cli gaps), 6 (testing value), 7 (PR validation), 11 (codification).
**Rubric:** `.claude/rules/repo-audit.md`. 59 raw findings deduplicated to **54**
(2 high / 15 medium / 37 low, including 17 positive/FYI records).

## Verdict

Repo quality is sound at the gate level and the currency-anchor system introduced in the
2026-07 sweeps is working -- most product-currency drift this cycle is incremental (new
features to document) rather than correctional (wrong claims to retract). Three themes
need action:

- **HIGH -- Snowflake converter rewiring is the single largest codification debt (2 high)** --
  both ts-convert-from-snowflake-sv and ts-convert-to-snowflake-sv have deterministic CLI
  commands shipped (`ts snowflake parse-sv` / `translate-formulas` / `build-model` /
  `build-sv`) but neither SKILL.md references them. The Databricks equivalents were fully
  rewired in BL-063 PR4 (v1.8.0, 2026-07-10); the Snowflake half remains BL-063 Phases
  1a--1c OPEN. This is the highest-value codification target by execution volume and the
  root cause of most cross-skill prose duplication (Spotter enablement step, post-import
  verification, common import error table all live inline because the skills bypass
  build-model).
- **MEDIUM -- Product-currency drift is completeness gaps, not broken claims (7 medium
  across 4 platforms)** -- Snowflake `cortex_search_service` changed from string to object
  in the YAML spec; Databricks added wildcard field expressions and statistical aggregates
  (median/percentile); Tableau has 6 date functions and 7 window/rank functions outside the
  mapping; ThoughtSpot's PARTIAL_OBJECT import policy is undocumented. None cause converter
  failures today but all represent gaps the next user or sweep will trip over.
- **MEDIUM -- Code duplication in ts-cli and cross-skill prose has reached extraction
  threshold (8 medium)** -- profile loading logic (4 call sites), JSON file-load helpers
  (4 modules), Snowflake connector construction (smoke tests vs load.py), and SKILL.md
  prose (connection selection prompt x4, table discovery x4, CLI-first rule x5, post-import
  verification x2) all exceed the "2+ sites" trigger for shared extraction.
